# Tasks

Active, scoped task definitions live here. One file per task.

A task file should contain:

- **Goal** — single sentence
- **Definition of done** — observable, testable
- **Inputs** — what is required to start
- **Outputs** — what gets produced
- **Owner** — who or which specialist handles it
- **Status** — not started / in progress / blocked / done
- **Notes** — anything else worth carrying

When a task is complete, move it to `docs/completed/` or delete it
after capturing any lesson in `memory/lessons.md`.
